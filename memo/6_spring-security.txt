▷ Spring security 
	애플리케이션의 인증과 인가 기능을 제공하는 Spring 의 하위 프레임 워크이다.

▶ 인증
	사용자의 계정을 확인하는 과정, 아이디와 비밀번호를 입력하는 인증을 진행한다.
	ex) 기본 인증, 폼 인증, OAuth, JWT 등이 있다.

▶ 인가 
	인증된 사용자가 특정 기능에 접근할 수 있는 권한을 확인하는 과정이다. 

▶ 필터 체인 (Filter chain)
	- 여러 보안 필터가 체인 형태로 연결되어 요청을 처리한다.
	- 각 필터는 특정 보안 기능을 담당한다. (인증 필터, 인가 필터, 세션 관리 필터 등등 이 있다.)

▶ 과정 
	① 요청이 들어온다
		↓
	② AuthenticationFilter 를 받는다 
		↓
	③ manager 가 인증 절차를 위임 받는다 
		↓
 	④ Provider 로 넘어가서, 실질적인 인증 절차인 Service 가 실행된다. 
		↓
	⑤ 반환된 정보를 가지고 2번 필터(AuthenticationFilter)로 다시 돌아간다. 
		↓
	⑥ SecurityContextHolder 가 받아서 마무리한다.
